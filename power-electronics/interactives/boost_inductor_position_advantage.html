Where the inductor sits determines what the input sees
The inductor on the input doesn't just filter current — it physically prevents instantaneous current changes. No matter how fast the MOSFET switches, the input terminal can only see smooth ramps. This is a topological guarantee, not a design choice you can accidentally undo.
Zoomed in — one switching cycle, showing what happens to input current
The MOSFET switches hard at 100kHz — but the input current never knows
The MOSFET current (red) has hard vertical edges — infinite dI/dt. But that discontinuity never reaches the input terminal because the inductor sits between them. The input only sees the triangular envelope (green), which has gentle, finite slopes set by Vin/L and (Vout−Vin)/L.
The chain of reasoning — why input-side L enables PFC
1
L on input → input current is always continuous
Current through L cannot jump. So even when the MOSFET switches at 100 kHz, the current flowing from the AC source into the inductor is a smooth ramp — never a step, never a discontinuity. This is a physical law, not a styling choice.
2
Continuous current → controllable average current
Because the current ramps linearly each switching cycle, the average current per cycle is the midpoint of the ramp. By controlling duty cycle, the controller sets that average current cycle by cycle.
3
Controllable average → shapeable waveform
If you can set the average current each cycle, you can make it follow a sine reference. A PFC controller adjusts duty cycle many thousands of times per second so each cycle average lands on the desired 50/60 Hz current waveform.
4
Sinusoidal current in phase with voltage → PF ≈ 1
With current shaped as a sine wave that tracks the AC voltage, the supply looks closer to a pure resistor to the grid. Real power approaches apparent power, and both phase displacement and harmonic distortion drop.
5
Why other topologies can’t do this as naturally
In a flyback or buck, the input current can be discontinuous at the input terminal. You can still add filtering or control, but the boost topology starts with a valuable property: the input-side inductor naturally smooths and exposes the current you want to shape.
Connecting back to your passive PFC experiment
Your passive inductor was also on the input side — same position. It also limited dI/dt and spread the current pulse. The difference is that without a control loop, the duty cycle isn't being adjusted to shape the current into a sine. The passive inductor just slows things down blindly. Active boost PFC uses the same physical principle — input-side inductor limiting dI/dt — but adds a control loop that exploits that controllability to sculpt the current into a precise sine reference. Same physics, vastly smarter exploitation of it.
